# ENGLISH
# Data Monitoring Project – General Documentation

## 1. Objective

The objective of this project is to build a data monitoring system in order to ensure data quality, reliability, and consistency over time.

The project focuses on three main aspects:

- Anomaly detection  
- Pipeline automation  
- Data drift & monitoring  

---

## 2. Dataset

The dataset used in this project is:

- Wind Power Generation Data – Forecasting (Kaggle)

Description:  
This dataset contains wind energy data collected from four different locations. It includes meteorological variables and power generation values.

---

## 3. Selected variables

The following variables were selected for the analysis:

- Time  
- Temperature  
- RelativeHumidity_2m  
- WindSpeed_100m  
- Power  

Objective:  
These variables were chosen because they directly impact wind power generation and are relevant for monitoring and analysis.

---

## 4. Project organization

The project is carried out as a team, with each member responsible for a specific part:

- Anomaly Detection  
- Pipeline Automation  
- Data Drift & Monitoring  

Objective:  
Divide the work while maintaining a coherent global system.

---

## 5. GitHub organization

The project is organized using GitHub:

- Each member works on their own branch  
- The `main` branch contains the final cleaned dataset  
- Each branch is merged into the main branch  

Objective:  
Ensure collaboration while keeping a stable and shared version of the dataset.

---

## 6. Data preprocessing

Before analysis, the dataset was prepared:

- Handling missing values  
- Selecting relevant variables  
- Formatting data (time, structure)  
- Ensuring consistency  

Objective:  
Prepare clean and usable data for each part of the project.

---

## 7. Global objective of the system

The system aims to:

- Detect anomalies in data  
- Automate data processing  
- Monitor data drift over time  

Objective:  
Build a reliable and scalable data monitoring system.

---

## 8. Project structure

This document presents a general overview of the project.

Each part is detailed in separate documentation:

- Anomaly detection documentation  
- Pipeline automation documentation  
- Data drift & monitoring documentation  

---

## 9. Perspectives

The next steps include:

- Improving automation of the pipeline  
- Enhancing monitoring tools  
- Integrating results into a dashboard

# FRANÇAIS
# Projet de Monitoring des Données – Documentation Générale

## 1. Objectif

L’objectif de ce projet est de mettre en place un système de monitoring des données afin d’assurer leur qualité, leur fiabilité et leur cohérence dans le temps.

Le projet se concentre sur trois axes principaux :

- Détection d’anomalies  
- Automatisation des pipelines  
- Data drift & monitoring  

---

## 2. Dataset

Le dataset utilisé dans ce projet est :

- Wind Power Generation Data – Forecasting (Kaggle)

Description :  
Ce dataset contient des données d’énergie éolienne provenant de quatre localisations différentes. Il inclut des variables météorologiques et des données de production d’énergie.

---

## 3. Variables sélectionnées

Les variables suivantes ont été sélectionnées pour l’analyse :

- Time  
- Temperature  
- RelativeHumidity_2m  
- WindSpeed_100m  
- Power  

Objectif :  
Ces variables ont été choisies car elles influencent directement la production d’énergie éolienne et sont pertinentes pour le monitoring.

---

## 4. Organisation du projet

Le projet est réalisé en équipe, avec une répartition des tâches :

- Détection d’anomalies  
- Automatisation des pipelines  
- Data drift & monitoring  

Objectif :  
Répartir le travail tout en gardant une cohérence globale.

---

## 5. Organisation GitHub

Le projet est organisé avec GitHub :

- Chaque membre travaille sur sa propre branche  
- La branche `main` contient le dataset final nettoyé  
- Les branches sont fusionnées dans la branche principale  

Objectif :  
Permettre un travail collaboratif tout en conservant une version stable des données.

---

## 6. Préparation des données

Avant l’analyse, les données ont été préparées :

- Gestion des valeurs manquantes  
- Sélection des variables pertinentes  
- Formatage des données (temps, structure)  
- Vérification de la cohérence  

Objectif :  
Obtenir des données propres et exploitables.

---

## 7. Objectif global du système

Le système vise à :

- Détecter des anomalies dans les données  
- Automatiser le traitement des données  
- Surveiller les dérives (data drift)  

Objectif :  
Construire un système de monitoring fiable et évolutif.
